## 1. はじめに

### ツールの概要

CSV Mapping Tool は、CSVファイル（表形式のデータファイル）のカラム構成やデータ内容を、別のフォーマットに変換するためのWebアプリケーションです。

たとえば、あるシステムから出力されたCSVファイルを、別のシステムに取り込める形式に変換したいときに活用できます。プログラミングの知識は不要で、画面上の操作だけで変換ルールを設定できます。

### 対象ユーザー

- 業務でCSVファイルを扱う担当者
- システム間のデータ移行を行う方
- ECサイトの商品データを整備する方
- 定期的にCSVのフォーマット変換が必要な方

### 用語の説明

| 用語 | 意味 |
|------|------|
| カラム | CSVファイルの「列」のこと。ヘッダー行の各項目名に相当します |
| マッピング | 変換前のカラムと変換後のカラムの対応付けのこと |
| コンバーター | データの変換ルール。文字の置換や分割などの加工方法を指します |
| プロファイル | マッピングやコンバーターの設定一式を保存したもの |
| エンコーディング | 文字コードのこと。日本語CSVでは UTF-8 または SJIS（Shift_JIS）がよく使われます |

## 2. 基本的な使い方

CSV Mapping Tool での作業は、以下の流れで進めます。

```
CSVファイルの読み込み → ターゲットカラムの設定 → マッピングの作成 → プレビューで確認 → CSVエクスポート
```

### 2.1 CSVファイルの読み込み

1. 画面上部の「変換前カラム」セクションにある **エンコーディング切り替えボタン** で、読み込むCSVファイルの文字コードを選択します
   - **UTF-8**: 一般的な文字コード。迷ったらこちらを選んでください
   - **SJIS**: Windows環境で作成されたCSVファイルでよく使われる文字コードです。文字化けする場合はこちらに切り替えてください
2. **「カラムインポート」ボタン** をクリックし、変換元のCSVファイルを選択します
3. 読み込みが成功すると、「変換前カラム」にCSVファイルのヘッダー行の項目名が一覧表示されます

また、画面上部の **「CSVインポート」ボタン** からCSVファイルを読み込むと、カラム定義とデータの両方が同時に読み込まれます。

### 2.2 ターゲットカラム（変換後カラム）の設定

変換後のCSVファイルに含めたいカラムを設定します。方法は2つあります。

**方法A: 変換前カラムからコピーする**

- 各カラム名の右にある **矢印ボタン（>）** をクリックすると、そのカラムが変換後カラムに追加されます
- **「全てコピー」ボタン** をクリックすると、変換前の全カラムを一括で変換後カラムにコピーできます

**方法B: 手動で追加する**

- 「変換後カラム」セクションの **「カラム追加」ボタン** をクリックします
- ダイアログにカラム名を入力して「保存」をクリックします
- 追加したカラムは、名前の編集や削除も可能です

### 2.3 マッピングの作成

マッピングとは、「変換前のどのカラムを、変換後のどのカラムに対応させるか」という設定です。

1. 「マッピング設定」セクションの **「マッピング追加」ボタン** をクリックします
2. マッピング行が追加されるので、以下を設定します
   - **変換元**: ドロップダウンから変換前のカラムを選びます（複数選択可能）
   - **変換先**: ドロップダウンから変換後のカラムを選びます
   - **コンバーター**: データの変換方法を選びます（詳細は[第3章](#3-コンバーター詳細)を参照）

**複数カラムの結合**

変換元で複数のカラムを選択すると、それらの値が結合されて変換先に出力されます。結合時の区切り文字は「区切り文字」欄で指定できます（空欄の場合はそのまま連結されます）。

例: 「姓」と「名」を選択し、区切り文字に半角スペースを設定すると、「山田 太郎」のように結合されます。

**自動マッピング**

変換前と変換後に同じ名前のカラムがある場合、**「同名カラム一括設定」ボタン** をクリックすると、一致するカラム同士のマッピングが自動的に作成されます。

### 2.4 プレビューと確認

画面下部の「プレビュー」セクションで、変換結果を確認できます。

- **「変換前」タブ**: 読み込んだCSVデータの元の内容を表示します
- **「変換後」タブ**: マッピングとコンバーターを適用した後のデータを表示します

プレビューを確認しながら、マッピングやコンバーターの設定を調整できます。設定を変更すると、プレビューはリアルタイムに更新されます。

### 2.5 CSVエクスポート

変換結果に問題がなければ、CSVファイルとしてダウンロードします。

1. プレビューセクション右上の **「CSVダウンロード」ボタン** をクリックします
2. 「converted.csv」というファイル名でダウンロードされます

データフィルター（後述）が有効な場合は、フィルター条件に一致した行のみがエクスポートされます。

## 3. コンバーター詳細

コンバーターは、変換元カラムの値を加工するための機能です。全11種類あり、必要に応じて組み合わせて使用できます。

### 3.1 そのまま（direct）

変換元の値をそのまま変換先にコピーします。何も加工しない場合に使います。

| 設定項目 | なし |
|----------|------|

| 入力 | 出力 |
|------|------|
| `東京都新宿区` | `東京都新宿区` |

### 3.2 分割（split）

テキストを指定した区切り文字で分割し、指定した位置の部分を取り出します。

| 設定項目 | 説明 |
|----------|------|
| 区切り文字 | テキストを分割する文字を指定します（例: スペース、カンマ、ハイフンなど） |
| インデックス | 取り出す位置を数値で指定します。0が最初の部分です。負の値を指定すると末尾から数えます |

| 入力 | 区切り文字 | インデックス | 出力 |
|------|-----------|-------------|------|
| `山田 太郎` | （半角スペース） | `0` | `山田` |
| `山田 太郎` | （半角スペース） | `1` | `太郎` |
| `2025-01-15` | `-` | `0` | `2025` |
| `A-B-C` | `-` | `-1` | `C` |

### 3.3 置換（replace）

テキスト内の特定の文字列を、別の文字列に置き換えます。該当する箇所がすべて置換されます。

| 設定項目 | 説明 |
|----------|------|
| 検索文字列 | 置き換えたい文字列を指定します |
| 置換後 | 置き換え先の文字列を指定します。空にすると、検索文字列が削除されます |

| 入力 | 検索文字列 | 置換後 | 出力 |
|------|-----------|--------|------|
| `03-1234-5678` | `-` | （空） | `0312345678` |
| `2025/01/15` | `/` | `-` | `2025-01-15` |
| `株式会社ABC` | `株式会社` | `(株)` | `(株)ABC` |

### 3.4 接頭辞付与（prefix）

テキストの先頭に文字列を追加します。3つのタイプから選べます。

| 設定項目 | 説明 |
|----------|------|
| タイプ | 付与する接頭辞の種類を選択します |

**固定文字列タイプ**

指定した文字列をそのまま先頭に追加します。

| 追加設定 | 説明 |
|----------|------|
| 接頭辞 | 先頭に追加する文字列 |

| 入力 | 接頭辞 | 出力 |
|------|--------|------|
| `12345` | `ID-` | `ID-12345` |
| `商品A` | `【新】` | `【新】商品A` |

**ランダムタイプ**

ランダムな英数字を先頭に追加します。行ごとに異なる値が生成されます。

| 追加設定 | 説明 |
|----------|------|
| 文字数 | 生成するランダム文字列の長さ（1〜32文字） |

| 入力 | 文字数 | 出力（例） |
|------|--------|-----------|
| `商品A` | `8` | `aB3kX9mP商品A` |

**日付タイプ**

現在の日時を指定したフォーマットで先頭に追加します。

| 追加設定 | 説明 |
|----------|------|
| フォーマット | 日付のフォーマット。使用可能なプレースホルダ: `YYYY`（年）、`MM`（月）、`DD`（日）、`HH`（時）、`mm`（分）、`ss`（秒） |

| 入力 | フォーマット | 出力（例） |
|------|-------------|-----------|
| `_data.csv` | `YYYYMMDD` | `20260328_data.csv` |
| `-report` | `YYYY-MM-DD_` | `2026-03-28_-report` |

### 3.5 接尾辞付与（suffix）

テキストの末尾に文字列を追加します。

| 設定項目 | 説明 |
|----------|------|
| 接尾辞 | 末尾に追加する文字列 |

| 入力 | 接尾辞 | 出力 |
|------|--------|------|
| `1000` | `円` | `1000円` |
| `山田` | `様` | `山田様` |
| `document` | `.pdf` | `document.pdf` |

### 3.6 トリム（trim）

テキストの前後にある空白文字（スペース、タブなど）を除去します。

| 設定項目 | 説明 |
|----------|------|
| トリム位置 | 空白を除去する位置を選択します |

| 選択肢 | 説明 |
|--------|------|
| 両端 | 先頭と末尾の両方の空白を除去します |
| 先頭のみ | 先頭の空白のみ除去します |
| 末尾のみ | 末尾の空白のみ除去します |

| 入力 | トリム位置 | 出力 |
|------|-----------|------|
| `  東京都  ` | 両端 | `東京都` |
| `  東京都  ` | 先頭のみ | `東京都  ` |
| `  東京都  ` | 末尾のみ | `  東京都` |

### 3.7 大文字/小文字（case）

アルファベットの大文字・小文字を変換します。

| 設定項目 | 説明 |
|----------|------|
| 変換タイプ | 変換の種類を選択します |

| 選択肢 | 説明 |
|--------|------|
| 大文字 | すべてのアルファベットを大文字に変換します |
| 小文字 | すべてのアルファベットを小文字に変換します |
| 先頭大文字 | 各単語の先頭文字を大文字、それ以外を小文字に変換します |

| 入力 | 変換タイプ | 出力 |
|------|-----------|------|
| `hello world` | 大文字 | `HELLO WORLD` |
| `Hello World` | 小文字 | `hello world` |
| `hello world` | 先頭大文字 | `Hello World` |

### 3.8 部分抽出（substring）

テキストの一部分だけを取り出します。

| 設定項目 | 説明 |
|----------|------|
| 開始位置 | 取り出しを開始する位置（0から数えます。0が先頭です） |
| 終了位置 | 取り出しを終了する位置（省略すると末尾まで取り出します） |

| 入力 | 開始位置 | 終了位置 | 出力 |
|------|---------|---------|------|
| `20250115` | `0` | `4` | `2025` |
| `20250115` | `4` | `6` | `01` |
| `20250115` | `6` | （空） | `15` |
| `ABC-12345` | `4` | （空） | `12345` |

### 3.9 パディング（padding）

テキストが指定した桁数に満たない場合、指定した文字で埋めます。商品コードや注文番号など、桁数を揃えたい場合に便利です。

| 設定項目 | 説明 |
|----------|------|
| 位置 | 埋める位置。「先頭」または「末尾」を選択します |
| 文字 | 埋めるために使う文字（1文字）。初期値は `0` です |
| 桁数 | 出力したい文字数の合計 |

| 入力 | 位置 | 文字 | 桁数 | 出力 |
|------|------|------|------|------|
| `42` | 先頭 | `0` | `5` | `00042` |
| `7` | 先頭 | `0` | `3` | `007` |
| `AB` | 末尾 | `_` | `5` | `AB___` |
| `12345` | 先頭 | `0` | `5` | `12345`（既に5桁なのでそのまま） |

### 3.10 条件代入（conditional）

別のカラムの値を条件として判定し、条件に一致するかどうかで出力する値を切り替えます。正規表現（テキストのパターンマッチング）を使って条件を指定します。

| 設定項目 | 説明 |
|----------|------|
| 条件カラム | 条件判定に使用するカラムを選択します |
| 正規表現パターン | 条件カラムの値と照合するパターン。正規表現（テキストの検索パターンを記述する書式）で指定します |
| マッチ時の値 | パターンに一致した場合に出力する値 |
| 非マッチ時の値 | パターンに一致しなかった場合に出力する値（省略すると変換元の値がそのまま使われます） |

**正規表現の簡単な例:**

| パターン | 意味 |
|----------|------|
| `東京` | 「東京」という文字列を含む |
| `^A` | 先頭が「A」で始まる |
| `\d{8}` | 数字が8桁連続する |
| `C\d{8}` | 「C」の後に数字が8桁続く |

| 変換元の値 | 条件カラムの値 | パターン | マッチ時 | 非マッチ時 | 出力 |
|-----------|--------------|----------|---------|-----------|------|
| `商品A` | `C00000001` | `C\d{8}` | `対象` | `対象外` | `対象` |
| `商品B` | `X999` | `C\d{8}` | `対象` | `対象外` | `対象外` |
| `既存値` | `東京都新宿区` | `東京` | `関東` | （空） | `関東` |
| `既存値` | `大阪府大阪市` | `東京` | `関東` | （空） | `既存値` |

### 3.11 日付フォーマット（dateFormat）

日付の表記形式を変換します。たとえば「2025年01月15日」を「2025-01-15」に変換できます。

| 設定項目 | 説明 |
|----------|------|
| 入力フォーマット | 変換元の日付がどのような形式で書かれているかを指定します |
| 出力フォーマット | 変換後の日付をどのような形式で出力するかを指定します |
| 月オフセット | 月をずらす値。例: `-1` で1ヶ月前、`1` で1ヶ月後 |

**使用できるプレースホルダ:**

| プレースホルダ | 意味 | 例 |
|---------------|------|-----|
| `YYYY` | 年（4桁） | `2025` |
| `MM` | 月（2桁） | `01` |
| `DD` | 日（2桁） | `15` |
| `hh` | 時（2桁） | `09` |
| `mm` | 分（2桁） | `30` |
| `ss` | 秒（2桁） | `45` |

| 入力 | 入力フォーマット | 出力フォーマット | 月オフセット | 出力 |
|------|-----------------|-----------------|-------------|------|
| `2025年01月15日` | `YYYY年MM月DD日` | `YYYY-MM-DD` | `0` | `2025-01-15` |
| `2025/01/15` | `YYYY/MM/DD` | `YYYYMMDD` | `0` | `20250115` |
| `01/15/2025` | `MM/DD/YYYY` | `YYYY-MM-DD` | `0` | `2025-01-15` |
| `2025-03-31` | `YYYY-MM-DD` | `YYYY-MM-DD` | `-1` | `2025-02-28` |
| `2025-01-15` | `YYYY-MM-DD` | `YYYY-MM-DDThh:mm:ssZ` | `0` | `2025-01-15T00:00:00+09:00` |

> **補足**: 入力に時刻情報が含まれていない場合、出力フォーマットの末尾の `Z` は自動的に `+09:00`（日本標準時）に変換されます。

> **補足**: 月オフセットを使用した際、日が変換先の月の最終日を超える場合は自動的にその月の末日に調整されます（例: 3月31日の1ヶ月前は2月28日）。

## 4. 便利な機能

### 4.1 プロファイル保存/復元

マッピング設定やコンバーターの構成を「プロファイル」として保存できます。同じ変換を繰り返し行う場合に便利です。

**保存される内容:**
- 変換前カラムの定義
- 変換後カラムの定義
- すべてのマッピング設定（コンバーター含む）
- エンコーディング設定
- データフィルター設定

**操作方法:**

1. 画面上部のヘッダーにある **「プロファイル」ボタン** をクリックします
2. メニューから以下の操作を選択できます

| 操作 | 説明 |
|------|------|
| 新規保存 | 現在の設定に名前を付けて新しいプロファイルとして保存します |
| 上書き保存 | 現在選択中のプロファイルを最新の設定で上書きします |
| インポート | JSONファイルからプロファイルを読み込みます |
| エクスポート | 保存済みプロファイルをJSONファイルとしてダウンロードします |
| 名前変更 | プロファイルの名前を変更します |
| 削除 | 不要なプロファイルを削除します |

保存されたプロファイルはブラウザのローカルストレージ（ブラウザ内部の保存領域）に格納されます。同じブラウザで次回アクセスしたときにもそのまま使えます。

> **注意**: ブラウザのデータを消去するとプロファイルも削除されます。大切な設定はJSONファイルとしてエクスポートしておくことをおすすめします。

**プロファイルの共有**

エクスポートしたJSONファイルを他の担当者に渡し、インポートしてもらうことで、同じ変換設定を共有できます。

### 4.2 データフィルター

プレビューおよびエクスポート時に、特定の条件に一致する行だけを抽出できます。

**設定方法:**

1. プレビューセクションにある **「データフィルター」スイッチ** をオンにします
2. 以下を設定します

| 設定項目 | 説明 |
|----------|------|
| 対象（変換前/変換後） | フィルター条件を変換前データと変換後データのどちらに適用するかを選択します |
| カラム | 条件判定に使うカラムを選択します |
| 正規表現パターン | 抽出条件をパターンで指定します |

フィルターが有効な場合、プレビュー画面に一致した件数が「(一致件数 / 全件数 件)」の形式で表示されます。

**CSVダウンロード時にもフィルターが適用されます。** フィルターが有効な状態でエクスポートすると、条件に一致した行だけが出力されます。

### 4.3 ドラッグ&ドロップ並び替え

以下の項目はドラッグ&ドロップ（マウスでつかんで移動する操作）で順序を変更できます。

- **変換後カラム**: カラム一覧の左側にあるドラッグハンドル（6つの点のアイコン）をつかんで上下に移動できます。エクスポートされるCSVのカラム順序はこの並び順になります
- **マッピング行**: マッピング設定の左側にあるドラッグハンドルをつかんで上下に移動できます。マッピングの適用順序を変更したい場合に使います

### 4.4 コンバーターの連鎖（パイプライン）

1つのマッピングに対して、複数のコンバーターを順番に適用できます。前のコンバーターの出力が次のコンバーターの入力になります。

**設定方法:**

1. マッピング行の「コンバーター」セクションにある **「+」ボタン** をクリックします
2. 新しいコンバーターが追加されます
3. 各コンバーターに番号が付いており、1番から順に処理されます
4. 不要なコンバーターは右側の削除ボタンで削除できます（コンバーターが2つ以上ある場合のみ）

**使用例: 電話番号のハイフンを除去してから桁数を揃える**

```
入力: "3-1234-5678"
  ↓ コンバーター1: 置換（"-" → ""）
中間結果: "312345678"
  ↓ コンバーター2: パディング（先頭、"0"、11桁）
出力: "0312345678"
```

**使用例: 名前の空白を除去して接尾辞を追加する**

```
入力: "  山田太郎  "
  ↓ コンバーター1: トリム（両端）
中間結果: "山田太郎"
  ↓ コンバーター2: 接尾辞付与（"様"）
出力: "山田太郎様"
```

### 4.5 複数カラムの結合

1つのマッピングで、変換元に複数のカラムを選択できます。選択した順にカラムの値が結合されます。

複数カラムを選択すると「区切り文字」入力欄が表示されます。ここに文字を指定すると、各カラムの値の間にその文字が挿入されます。

**使用例: 住所の結合**

変換元: 「都道府県」「市区町村」「番地」を選択、区切り文字は空欄

```
都道府県: "東京都"、市区町村: "新宿区"、番地: "1-2-3"
→ 出力: "東京都新宿区1-2-3"
```

## 5. よくある使い方

### 5.1 ECサイト商品CSVのフォーマット変換

あるECプラットフォームの商品データCSVを、別のプラットフォーム向けのCSVに変換する例です。

**想定シナリオ:** プラットフォームAの商品CSVを、プラットフォームBの形式に変換する

1. プラットフォームAからエクスポートしたCSVを読み込みます
2. プラットフォームBが要求するカラム名を変換後カラムに手動で追加します
3. マッピングを設定します
   - 「商品名」→「product_name」: そのまま
   - 「価格（税込）」→「price」: 置換（「,」→「」）で桁区切りのカンマを除去
   - 「商品コード」→「sku」: 大文字/小文字（大文字）で英字を統一
   - 「カテゴリ > サブカテゴリ」→「category」: 分割（「>」、インデックス 0）でメインカテゴリのみ取得し、トリムで空白を除去（コンバーター連鎖）
4. プレビューで確認し、CSVダウンロード

設定を「プラットフォームB変換」などの名前でプロファイル保存しておくと、次回以降は同じ設定を読み込むだけで変換できます。

### 5.2 システム間データ移行

旧システムから新システムへデータを移行する例です。

**想定シナリオ:** 顧客管理システムのCSVデータを新システムの形式に変換する

1. 旧システムからエクスポートしたCSV（SJIS）を読み込みます
   - エンコーディングを「SJIS」に切り替えてからインポートします
2. 新システムのカラム構成を変換後カラムに設定します
3. マッピングを設定します
   - 「顧客番号」→「customer_id」: パディング（先頭、"0"、10桁）で桁数を統一
   - 「姓」「名」→「full_name」: 複数カラム選択で結合（区切り文字: 半角スペース）
   - 「電話番号」→「tel」: 置換（「-」→「」）でハイフンを除去
   - 「登録日」→「registered_at」: 日付フォーマット（入力: `YYYY/MM/DD`、出力: `YYYY-MM-DDThh:mm:ssZ`）でISO 8601形式に変換
   - 「区分」→「type」: 条件代入で値を変換（パターン: `^法人$`、マッチ時: `corporate`、非マッチ時: `individual`）
4. データフィルターで特定の顧客区分のみ抽出してエクスポートすることも可能です

### 5.3 日付フォーマットの統一

複数のシステムから出力されたCSVの日付形式がバラバラな場合に、統一する例です。

**想定シナリオ:** 日付形式を `YYYY-MM-DD` に統一する

各CSVファイルに対して、日付フォーマットコンバーターを設定します。

| 元の形式 | 入力フォーマット | 出力フォーマット |
|---------|-----------------|-----------------|
| `2025年01月15日` | `YYYY年MM月DD日` | `YYYY-MM-DD` |
| `01/15/2025` | `MM/DD/YYYY` | `YYYY-MM-DD` |
| `20250115` | `YYYYMMDD` | `YYYY-MM-DD` |

**月オフセットの活用例:**

請求書の「利用月」が翌月表記になっているデータを当月に修正する場合、月オフセットに `-1` を設定します。

| 入力 | 入力フォーマット | 出力フォーマット | 月オフセット | 出力 |
|------|-----------------|-----------------|-------------|------|
| `2025-04-30` | `YYYY-MM-DD` | `YYYY-MM-DD` | `-1` | `2025-03-30` |
| `2025-03-31` | `YYYY-MM-DD` | `YYYY-MM-DD` | `-1` | `2025-02-28` |

プロファイルとして保存しておけば、毎月の定型作業を効率化できます。
