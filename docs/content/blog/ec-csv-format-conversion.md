# 楽天の商品CSVをYahoo!ショッピング形式に変換したい――EC多店舗運用のCSV地獄から抜け出す方法

## はじめに：EC担当者なら誰もが経験する「CSV変換作業」

「楽天からエクスポートした商品CSVを、Yahoo!ショッピングにアップロードしたい」

EC多店舗運営をしていると、この手の作業は避けて通れません。商品データの一元管理ツールを導入していればいいのですが、月額コストの問題や、取扱商品が少ないうちは「手作業でなんとかなるだろう」と思いがちです。

ところが実際にやってみると、これが想像以上に大変です。

- カラム名が違う（`商品名` vs `Title`、`販売価格` vs `price`）
- 日付のフォーマットが違う（`2026/03/28` vs `2026-03-28`）
- カテゴリの値体系がまるで違う
- 文字コードがSJISだったりUTF-8だったり

毎回Excelを開いて、VLOOKUP、IF関数、TEXT関数を駆使して変換している方も多いのではないでしょうか。

## 手作業・Excel関数の限界

Excelでの変換作業には、いくつかの根本的な問題があります。

**1. 毎回同じ作業を繰り返す**

商品を追加するたびに、あるいは月次の在庫更新のたびに、同じ列の入れ替えや関数のコピーを手動で行う必要があります。500行の商品データを毎週変換する場合、年間で膨大な時間が消えていきます。

**2. 関数が壊れやすい**

列を挿入・削除したときに参照がずれる。誰かがファイルを編集して関数が消える。「先月まで動いていたのに」という悲鳴は、EC運用の現場で日常茶飯事です。

**3. 条件分岐が複雑になりがち**

「カテゴリが"食品"なら"1001"、"衣類"なら"2001"に変換する」といったマッピングは、IFのネストが深くなり、可読性もメンテナンス性も急激に悪化します。

**4. 文字コード問題**

Yahoo!ショッピングのCSVはSJIS指定のケースがあります。UTF-8で作ったCSVをアップロードして文字化け、という経験をした方も少なくないでしょう。

## 解決策：CSV-Mapping Tool

こうしたCSV変換の課題を解決するために作られたのが **[CSV-Mapping Tool](https://tamoco-mocomoco.github.io/csv-mapping-tool/)** です。

主な特徴は以下のとおりです。

- **ブラウザ完結**: インストール不要。URLにアクセスするだけで使えます
- **無料**: 商用利用も含めて完全無料です
- **データは外部送信なし**: すべての処理がブラウザ内で完結するため、商品データが外部サーバーに送られることはありません
- **SJIS対応**: 読み込み・書き出しの両方でSJISエンコーディングに対応しています

### 基本的な使い方（3ステップ）

1. **変換元CSVを読み込む** -- 画面上部の「CSVインポート」からファイルを選択
2. **変換先のカラムを定義してマッピングを設定** -- どのカラムをどのカラムに対応させるかをドロップダウンで指定
3. **プレビューで確認してCSVダウンロード** -- 変換結果をリアルタイムで確認し、問題なければダウンロード

プログラミングの知識は一切不要です。

## 具体例：楽天 → Yahoo!ショッピング的な変換シナリオ

ここからは、ECモール間の商品CSV変換を想定した具体例を見ていきます。実際のモール仕様とは異なる部分がありますが、変換パターンの参考にしてください。

### シナリオ設定

変換元CSVのカラム構成（モールA形式）：

| 商品管理番号 | 商品名 | 販売価格 | カテゴリパス | 登録日 | 商品説明 |
|---|---|---|---|---|---|
| item-001 | オーガニックコーヒー 200g | 1,280 | 食品 > コーヒー > 豆 | 2026/03/15 | 厳選された... |

変換先CSVのカラム構成（モールB形式）：

| product_code | title | price | category_id | release_date | description |

カラム名も値のフォーマットも異なります。これを1つずつ変換していきましょう。

### 変換1：カラム名の変更（そのままコンバーター）

最もシンプルなケースです。`商品名` → `title` のように、カラム名が違うだけで値はそのまま使う場合は、コンバーターを「そのまま」に設定するだけです。

- `商品名` → `title`：そのまま
- `商品説明` → `description`：そのまま

### 変換2：価格のカンマ除去（置換コンバーター）

モールAの `販売価格` は `1,280` のように桁区切りカンマが入っていますが、モールBの `price` は数値のみを受け付けます。

**置換コンバーター**を使い、検索文字列に `,` を、置換後を空欄にすれば、`1,280` → `1280` に変換できます。

### 変換3：日付フォーマットの変更（日付フォーマットコンバーター）

モールAの `登録日` は `2026/03/15` 形式ですが、モールBの `release_date` は `2026-03-15` 形式を要求しています。

**日付フォーマットコンバーター**で以下のように設定します。

- 入力フォーマット：`YYYY/MM/DD`
- 出力フォーマット：`YYYY-MM-DD`

これだけで `2026/03/15` → `2026-03-15` に変換されます。

### 変換4：カテゴリ値の条件変換（条件代入コンバーター）

ここがExcelだと最も苦労するポイントです。モールAでは `食品 > コーヒー > 豆` というパス形式ですが、モールBではカテゴリIDでの指定が必要です。

**条件代入コンバーター**を使えば、正規表現パターンで柔軟に振り分けられます。

コンバーターを連鎖（パイプライン）させて、段階的に変換することも可能です。

- コンバーター1：条件代入（条件カラム = カテゴリパス、パターン = `^食品`、マッチ時 = `1001`、非マッチ時 = そのまま）
- コンバーター2：条件代入（条件カラム = カテゴリパス、パターン = `^衣類`、マッチ時 = `2001`、非マッチ時 = そのまま）
- コンバーター3：条件代入（条件カラム = カテゴリパス、パターン = `^家電`、マッチ時 = `3001`、非マッチ時 = そのまま）

パターンに `^食品` と指定すれば「食品」で始まるカテゴリパスすべてにマッチするので、サブカテゴリが異なっていても一括で変換できます。

### 変換5：商品コードの整形（大文字変換 + 接頭辞）

モールBでは商品コードを大文字英字で統一し、先頭に `MB-` を付ける必要がある場合。

- コンバーター1：大文字/小文字（大文字）→ `item-001` → `ITEM-001`
- コンバーター2：接頭辞付与（固定文字列 `MB-`）→ `ITEM-001` → `MB-ITEM-001`

このように、コンバーターの連鎖で複雑な変換も段階的に組み立てられます。

## まとめ：プロファイル保存で「次回からワンクリック」

ここまで設定したマッピングとコンバーターの構成は、**プロファイル**として名前を付けて保存できます。

たとえば「モールA → モールB 商品変換」という名前で保存しておけば、次回以降は以下の手順だけで変換が完了します。

1. プロファイルを選択して読み込む
2. 変換元CSVをインポートする
3. プレビューを確認してダウンロード

Excelのテンプレートファイルを管理する必要も、関数が壊れていないか毎回確認する必要もありません。

さらに、プロファイルはJSONファイルとしてエクスポートできるので、チーム内の他の担当者に共有することも簡単です。「この変換設定ファイルを読み込んで、あとはCSVを突っ込むだけ」と伝えるだけで引き継ぎが完了します。
