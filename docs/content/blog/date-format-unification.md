# CSVの日付フォーマットがバラバラ問題を一発解決する方法

## はじめに

業務でCSVデータを扱っていると、こんな経験はないでしょうか。

- 基幹システムからエクスポートしたCSVは `2024/03/15`
- SaaSの管理画面からダウンロードしたCSVは `2024-03-15T09:30:00Z`
- 取引先から届いたCSVは `2024年03月15日`

これらを1つのファイルにマージしようとした瞬間、日付フォーマットの不一致に気づいて手が止まる。Excelで開いて手動で直す？　VLOOKUPとTEXT関数を駆使する？　Pythonスクリプトを書く？

どれも「ちょっとした変換」にしては手間がかかりすぎます。

この記事では、ブラウザだけで完結するCSV変換ツール **CSV-Mapping Tool** の日付フォーマットコンバーターを使って、バラバラな日付フォーマットを一発で統一する方法を紹介します。

## よくある日付フォーマットのパターン

実務で遭遇する日付フォーマットを整理すると、おおむね以下のパターンに分類できます。

| パターン | 例 | 主な出所 |
|---|---|---|
| `YYYY/MM/DD` | 2024/03/15 | 日本の基幹系システム、Excel |
| `YYYY-MM-DD` | 2024-03-15 | データベース、API |
| `YYYY年MM月DD日` | 2024年03月15日 | 帳票系システム、PDF出力 |
| `DD/MM/YYYY` | 15/03/2024 | 欧州系システム |
| `MM/DD/YYYY` | 03/15/2024 | 米国系システム |
| `YYYY-MM-DDThh:mm:ssZ` | 2024-03-15T09:30:00Z | ISO 8601（API、ログ） |

厄介なのは、`01/02/2024` のようなデータが「1月2日」なのか「2月1日」なのか、フォーマットを知らなければ判別できないことです。フォーマットが混在したCSVをそのまま結合すると、データの意味が変わってしまうリスクがあります。

## CSV-Mapping Toolとは

**CSV-Mapping Tool** は、ブラウザ上で動作するCSV変換ツールです。

- **インストール不要** -- URLにアクセスするだけで使えます
- **データはローカル処理** -- サーバーにCSVデータを送信しません
- **無料・OSS** -- GitHubでソースコードを公開しています

URL: https://tamoco-mocomoco.github.io/csv-mapping-tool/

入力CSVのカラムごとに変換ルール（コンバーター）を設定し、出力CSVのフォーマットを自由に定義できます。日付フォーマットコンバーターはその機能の1つです。

## 日付フォーマットコンバーターの使い方

### 基本的な流れ

1. CSV-Mapping Toolにアクセスし、入力CSVをドラッグ&ドロップで読み込む
2. 日付カラムのコンバーターとして「日付フォーマット」を選択する
3. **入力フォーマット** と **出力フォーマット** を指定する
4. 変換を実行して出力CSVをダウンロードする

### フォーマット指定のプレースホルダ

日付フォーマットの指定には、以下のプレースホルダを使います。

| プレースホルダ | 意味 |
|---|---|
| `YYYY` | 年（4桁） |
| `MM` | 月（1〜2桁、自動ゼロ埋め） |
| `DD` | 日（1〜2桁、自動ゼロ埋め） |
| `hh` | 時（1〜2桁、自動ゼロ埋め） |
| `mm` | 分（1〜2桁、自動ゼロ埋め） |
| `ss` | 秒（1〜2桁、自動ゼロ埋め） |

### 変換の具体例

**例1: 和風フォーマットからISO形式へ**

```
入力フォーマット: YYYY年MM月DD日
出力フォーマット: YYYY-MM-DDThh:mm:ssZ
```

| 変換前 | 変換後 |
|---|---|
| 2024年11月30日 | 2024-11-30T00:00:00+09:00 |

入力に時刻情報がない場合、時刻は `00:00:00` で補完され、出力フォーマット末尾の `Z` は自動的に `+09:00`（JST）に置き換わります。日本のシステムで日付のみのデータを扱う場合に便利な仕様です。

**例2: スラッシュ区切りからハイフン区切りへ**

```
入力フォーマット: YYYY/MM/DD
出力フォーマット: YYYY-MM-DD
```

| 変換前 | 変換後 |
|---|---|
| 2024/1/5 | 2024-01-05 |

1桁の月・日は自動的にゼロ埋めされます。

**例3: 欧州式（DD/MM/YYYY）からISO形式へ**

```
入力フォーマット: DD/MM/YYYY
出力フォーマット: YYYY-MM-DD
```

| 変換前 | 変換後 |
|---|---|
| 30/11/2024 | 2024-11-30 |

入力フォーマットを正しく指定すれば、日と月が逆転している欧州式でも正確に変換できます。

**例4: 日時フォーマットの変換**

```
入力フォーマット: YYYY-MM-DD hh:mm:ss
出力フォーマット: YYYY/MM/DD hh時mm分ss秒
```

| 変換前 | 変換後 |
|---|---|
| 2024-01-15 09:30:45 | 2024/01/15 09時30分45秒 |

日付だけでなく、時刻を含むフォーマット変換にも対応しています。

## 応用: 月オフセットで「前月末日」を算出する

日付フォーマットコンバーターには **月オフセット** 機能があります。これは変換時に月を加算・減算できる機能で、実務で意外と重宝します。

### ユースケース: 請求締め日の算出

「毎月末日締め、翌月10日に届くCSV」を処理するケースを考えます。CSVの日付カラムには処理日（翌月の日付）が入っていますが、実際に必要なのは「前月の末日」です。

```
入力フォーマット: YYYY/MM/DD
出力フォーマット: YYYY-MM-DD
月オフセット: -1
```

| 変換前 | 変換後 |
|---|---|
| 2024/03/31 | 2024-02-29 |
| 2024/01/15 | 2023-12-15 |
| 2024/11/15 | 2024-10-15 |

注目すべきは `2024/03/31` の変換結果です。3月31日から1ヶ月前に戻すと「2月31日」になりますが、2024年の2月は29日まで（うるう年）なので、自動的に `2024-02-29` にクランプされます。年境界をまたぐケース（1月 → 前年12月）も正しく処理されます。

この「日のクランプ」の挙動は、月末日の算出にそのまま活用できます。たとえば、各月の1日のデータに対して月オフセット `-1` を適用すれば、前月の末日が自動的に算出されます。

## まとめ

CSVの日付フォーマット統一は、地味ながら多くの業務で繰り返し発生する作業です。CSV-Mapping Toolの日付フォーマットコンバーターを使えば、以下のメリットがあります。

- **プレースホルダ指定** で、あらゆる日付フォーマット間の変換に対応
- **自動ゼロ埋め** で、`2024/1/5` のような不揃いなデータも正規化
- **JST自動付与** で、日付のみのデータをISO 8601に変換する際のタイムゾーン指定が不要
- **月オフセット** で、前月末日の算出や締め日計算も1ステップ
- **ブラウザ完結** で、環境構築やデータのアップロードが不要

日付フォーマットの変換で消耗している方は、ぜひ試してみてください。
