# チャネル戦略: CSV-Mapping Tool

## 概要

CSV-Mapping Toolの認知拡大と利用者獲得のためのチャネル戦略を定義する。無料・OSSという特性を最大限に活かし、「使ってみたら便利だった」という体験起点で自然に広がる導線を設計する。

---

## チャネル優先度マトリクス

| 優先度 | チャネル | 期待効果 | 立ち上がりまでの期間 |
|---|---|---|---|
| **S（最優先）** | Qiita / Zenn | 技術者・業務担当者への直接リーチ | 1〜2週間 |
| **S（最優先）** | GitHub（README・Topics・Stars） | OSSとしての信頼性構築 | 1週間 |
| **A（高）** | 検索流入（SEO） | 課題ドリブンの自然流入 | 1〜3ヶ月 |
| **A（高）** | はてなブックマーク | バズによる一時的な大量流入 | 記事次第（不定） |
| **B（中）** | X（Twitter） | 継続的な認知維持、コミュニティ形成 | 2〜4週間 |
| **C（低〜将来）** | コミュニティ（Slack / Discord等） | 特定業種への深い浸透 | 1〜3ヶ月 |

---

## チャネル別の詳細戦略

### 1. Qiita / Zenn（技術記事）

**優先度: S（最優先）**

**なぜこのチャネルが重要か:**
- ペルソナ2（社内SE）が日常的に情報収集するプラットフォーム
- 「CSV 変換」「SJIS」など業務系の検索でQiita記事がGoogle上位に表示されやすい
- 記事がストック型コンテンツとなり、長期的に流入を生む

**具体的アクション:**

| # | アクション | 内容 | 投稿先 |
|---|---|---|---|
| 1 | ユースケース記事（EC編） | 「楽天・Amazon・Yahoo!ショッピングの商品CSVを一括変換する方法」 | Qiita |
| 2 | 技術解説記事 | 「ブラウザだけでCSV変換を完結させる設計思想 — PapaParse + encoding-japanese」 | Zenn |
| 3 | ユースケース記事（データ移行編） | 「システムリプレイスのCSVデータ移行をGUIだけで完結させた話」 | Qiita |
| 4 | How-to記事 | 「CSVの文字コード変換（SJIS↔UTF-8）をブラウザだけで安全に行う方法」 | Qiita |
| 5 | 比較記事 | 「ExcelでのCSV変換作業を専用ツールに置き換えてみた — 作業時間を1/3にした話」 | Zenn |

**記事構成のポイント:**
- タイトルに具体的な課題を含める（「楽天 CSV」「SJIS」等の検索キーワード）
- 冒頭で「この記事で解決できること」を3点に絞って提示
- スクリーンショットを多用し、操作手順を視覚的に示す
- 末尾に「すぐ試せるURL」を掲載し、即体験への導線を作る
- タグ: `CSV`, `データ変換`, `EC`, `OSS`, `業務効率化`

**期待効果:**
- 記事あたり 500〜3,000 PV（初月）
- Qiita記事がGoogle検索に露出し始めると、月間 1,000〜5,000 の継続流入
- 記事からツールURLへのクリック率 5〜10%

**KPI:**
- 記事公開数: 初月3本、以降月2本
- 記事からのツールアクセス数

### 2. GitHub（README・Stars獲得・Topics設定）

**優先度: S（最優先）**

**なぜこのチャネルが重要か:**
- OSSとしての信頼性・発見可能性の基盤
- GitHub Starsの数はツールの信頼指標として機能する
- Topics設定によりGitHub内検索で発見される
- README.mdが実質的なランディングページになる

**具体的アクション:**

| # | アクション | 詳細 |
|---|---|---|
| 1 | Topics設定 | `csv`, `csv-converter`, `csv-mapping`, `data-transformation`, `browser-tool`, `sjis`, `react`, `typescript`, `oss`, `japanese` |
| 2 | README強化（済） | デモGIF/スクリーンショットの追加、バッジ（ライセンス、デプロイステータス）の追加 |
| 3 | GitHub Pages のAbout欄 | Description: 「ブラウザ完結のCSVカラムマッピング＆変換ツール」、Website URLを設定 |
| 4 | CONTRIBUTING.md作成 | コントリビューションガイドを整備し、OSS参加の敷居を下げる |
| 5 | Releaseノート整備 | 機能追加時にGitHub Releasesを作成し、Watchersに通知を送る |
| 6 | Awesome CSV系リストへの掲載申請 | `awesome-csv` や `awesome-data-tools` 等のキュレーションリストにPRを送る |

**期待効果:**
- Topics設定によりGitHub Explore経由の発見
- Awesome系リスト掲載で安定的なStar獲得
- Star数が増えることで、記事・SNS投稿時の信頼性が向上

**KPI:**
- Stars数: 3ヶ月で50、6ヶ月で200を目標
- GitHub経由のPages アクセス数

### 3. 検索流入（SEO）

**優先度: A（高）**

**なぜこのチャネルが重要か:**
- ペルソナ1（EC運用担当者）は「楽天 CSV 変換」等で検索して解決策を探す
- ペルソナ3（データ移行担当者）は「CSV カラム変換 ツール」等で検索する
- 課題を持ったユーザーに直接リーチできる唯一のチャネル

**具体的アクション:**

| # | アクション | 詳細 |
|---|---|---|
| 1 | index.html meta tags最適化 | title: 「CSV-Mapping Tool - ブラウザ完結のCSVカラム変換ツール」、descriptionにキーワードを含める |
| 2 | OGP設定 | og:title, og:description, og:image を設定し、SNS共有時の見栄えを改善 |
| 3 | 構造化データ追加 | SoftwareApplicationスキーマをJSON-LDで埋め込み |
| 4 | ランディングページのテキスト追加 | GitHub Pagesのトップページに機能紹介テキストを追加（現在はアプリUIのみ） |
| 5 | ブログ/ドキュメントページの構築 | `/docs/` 配下にユースケース記事を公開し、SEOコンテンツとして機能させる |

**対象キーワード（詳細はseo-keywords.mdを参照）:**
- メイン: 「CSV変換ツール」「CSVマッピング」「CSVカラム変換」
- ロングテール: 「楽天 CSV フォーマット 変換」「CSV SJIS UTF-8 変換 ブラウザ」

**期待効果:**
- SEO施策開始から3ヶ月後に主要キーワードで検索結果の1〜3ページ目に表示
- 月間 2,000〜10,000 の検索流入（6ヶ月後目標）

**KPI:**
- 主要キーワードの検索順位
- オーガニック検索からのアクセス数

### 4. はてなブックマーク

**優先度: A（高）**

**なぜこのチャネルが重要か:**
- 日本語圏のIT系コンテンツにおいて最大のバイラル発生源
- 「3ブクマ → 新着エントリー → ホッテントリ」の流れに乗ると1日で数万PVが可能
- はてブからの流入はQiita/Zenn記事経由で発生しやすい

**具体的アクション:**

| # | アクション | 詳細 |
|---|---|---|
| 1 | バズを狙える記事設計 | 「Excel手作業を撲滅」系の共感を呼ぶタイトル設定 |
| 2 | 記事公開タイミング | 火曜〜木曜の午前中に公開（はてブの新着が活発な時間帯） |
| 3 | 初期ブクマの確保 | 記事公開時にX（Twitter）で告知し、フォロワーからのブクマを誘導 |
| 4 | セルフブックマーク | 自身で最初の1ブクマをつけ、コメント欄に概要を記載 |

**バズしやすい記事テーマ:**
- 「無料でここまでできる」系: 「無料のCSV変換ツールを作ったらEC運用担当者に感謝された話」
- 「脱Excel」系: 「Excel職人のCSV変換手順をツール化したら作業時間が1/3になった」
- 「セキュリティ」系: 「データを外部に送らずにCSV変換できるツールを作った理由」

**期待効果:**
- ホッテントリ入りした場合: 1日 10,000〜50,000 PV
- 新着エントリー止まりでも: 500〜2,000 PV

**KPI:**
- はてブ数
- はてブ経由のツールアクセス数

### 6. コミュニティ（Slack / Discord等）

**優先度: C（低〜将来）**

**なぜこのチャネルが重要か:**
- 特定の業種・職種に深く浸透できる
- ユーザーの生の課題を聞くことでプロダクト改善に活かせる
- 口コミの発生源になる

**具体的アクション:**

| # | アクション | 対象コミュニティ |
|---|---|---|
| 1 | EC系コミュニティへの参加・情報提供 | ECに関するSlack/Discordコミュニティ |
| 2 | 情シス系コミュニティでの紹介 | 情シスSlack、Corp-Eng Slack |
| 3 | OSS系コミュニティでの共有 | React / TypeScript系のDiscordサーバー |
| 4 | 質問回答による露出 | teratail、Stack Overflow（日本語版）でCSV関連の質問に回答する際にツールを紹介 |

**注意点:**
- 自己宣伝にならないよう、まず質問への回答や情報提供で価値を提供する
- 「こういうツールを使ったら解決しました」という体験共有の形式が効果的
- コミュニティのルールに従い、宣伝が許可されている場所でのみ紹介する

**期待効果:**
- 特定業種でのコアユーザー獲得
- 口コミによる自然拡散
- プロダクトフィードバックの獲得

**KPI:**
- コミュニティ経由のアクセス数
- コミュニティ内での言及数

## チャネル間の連携フロー

```
GitHub (信頼性基盤)
  ↕
Qiita/Zenn (記事公開)
  ↓ 告知                ↓ 検索露出
X (拡散) → はてブ (バイラル)   SEO (継続流入)
  ↓                            ↓
コミュニティ (深い浸透)      ツールURL (コンバージョン)
```

**連携の具体例:**
1. Qiita/Zennに記事を公開
2. X（Twitter）で記事の公開を告知
3. はてなブックマークに自身で最初のブクマをつける
4. 記事がバズれば、検索エンジンにもインデックスされやすくなる
5. ツールURLへの流入が増え、GitHub Starsも増加
6. Starsの増加が次の記事の信頼性を高める（好循環）
